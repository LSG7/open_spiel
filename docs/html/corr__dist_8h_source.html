27 // This file provides a set of functions for computing the distance of a
28 // distribution of joint policies to a correlated equilibrium. It is the
29 // analogue of NashConv for correlated equilibria, i.e. sum of the incentives to
30 // deviate to a best response over all players, as an empirical metric that
31 // summarizes how far the distribution is from an equilibrium.
32 //
33 // The functions compute these metrics for extensive-form correlated equilibria
34 // (EFCE) and extensive-form coarse-correlated equilibria (EFCCE). The
35 // algorithms work by constructing an auxiliary game (similar to the one
36 // described in Def 2.2 of von Stengel and Forges 2008) where chance initially
37 // samples a joint policy, then lets the players decide to follow or not follow
38 // the recommendations.
39 //
40 // The definition we use matches the common interpretation of EFCE's within AI
41 // papers which are based on causal deviations described in (Gordon, Greenwald,
42 // and Marks '08), (Dudik & Gordon '09), and (Farina & Sandholm '19).
43 // Specifically: if players follow recommendations, they continue to receive
44 // recommendations. If a player deviates, that player stops receiving
45 // recommendations from then on. The incentive for a player to deviate toward a
46 // best response can be computed by the existing best response algorithm in this
47 // new game.
48 //
49 // In both cases of EFCE and EFCCE, the algorithms compute the normal-form
50 // equivalents, and two wrapper functions are provided specifically for the
51 // normal-form setting (CEDist and CCEDist).
52 //
53 // **WARNING**: the implementations of the metrics for the extensive-form
54 // versions of the correlated equilibria have only been lightly tested (on a
55 // few simple examples). We plan to add more thorough tests as we implement
56 // more benchmark general-sum games and more worked-out examples.
57 //
58 // For formal definitions and algorithms, please refer to:
59 // - von Stengel and Forges, 2008. Extensive-Form Correlated Equilibrium:
60 // Definition and Computational Complexity, Mathematics of Operations
61 // Research, vol 33, no. 4.
62 // - Farina, Bianchi, and Sandholm, 2019. Coarse Correlation in Extensive-Form
63 // Games. https://arxiv.org/abs/1908.09893
64 // - Dudik & Gordon, https://arxiv.org/abs/1205.2649
65 // - Gordon, Greenwald, and Marks. No-Regret Learning in Convex Games.
66 // https://www.cs.cmu.edu/~ggordon/gordon-greenwald-marks-icml-phi-regret.pdf
67 
68 // A CorrelationDevice represents a distribution over joint policies (name is
69 // from von Stengel & Forges 2008). Note, however, that unlike von Stengel &
70 // Forges 2008, the joint policies can be mixed. In this case, an equivalent
71 // joint distribution over deterministic joint policies could be reconstructed
72 // (if the game is small enough) or the metrics below can be approximated via
73 // Monte Carlo sampling of deterministic joint policies from the mixtures.
74 using CorrelationDevice = std::vector<std::pair<double, TabularPolicy>>;

114 // Compute the sum of individual incentives to deviate (from the joint
115 // distribution) to a best response, over all players. The auxiliary game
116 // constructed is with accordance to the EFCE concept, which means players see
117 // their recommendations once they reach the information states (unless they've
118 // chosen not to follow at some point).
119 double EFCEDist(const Game& game, CorrDistConfig config,
120  const CorrelationDevice& mu);
121 
122 // Compute the sum of individual incentives to deviate (from the joint
123 // distribution) to a best response, over all players. The auxiliary game
124 // constructed is with accordance to the EFCCE concept, which means players see
125 // their recommendations at their information states only after they've decided
126 // whether or not to follow them.
127 double EFCCEDist(const Game& game, CorrDistConfig config,
128  const CorrelationDevice& mu);

130 // Agent-form variants: these are similar to EFCCE + EFCE distances above,
131 // except that there is at most one deviation allowed, at a single information
132 // set, but any information set. Other than this restriction, each one is
133 // analogous to EFCCE or EFCE.
134 // **Note: these have not yet been extensively tested.**
135 double AFCEDist(const Game& game, CorrDistConfig config,
136  const CorrelationDevice& mu);
