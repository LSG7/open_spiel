31 def __init__(self, game, average_type=AverageType.SIMPLE):
32 super().__init__(game)
33 # How to average the strategy. The 'simple' type does the averaging for
34 # player i + 1 mod num_players on player i's regret update pass; in two
35 # players this corresponds to the standard implementation (updating the
36 # average policy at opponent nodes). In n>2 players, this can be a problem
37 # for several reasons: first, it does not compute the estimate as described
38 # by the (unbiased) stochastically-weighted averaging in chapter 4 of
39 # Lanctot 2013 commonly used in MCCFR because the denominator (important
40 # sampling correction) should include all the other sampled players as well
41 # so the sample reach no longer cancels with reach of the player updating
42 # their average policy. Second, if one player assigns zero probability to an
43 # action (leading to a subtree), the average policy of a different player in
44 # that subtree is no longer updated. Hence, the full averaging does not
45 # update the average policy in the regret passes but does a separate pass to
46 # update the average policy. Nevertheless, we set the simple type as the
47 # default because it is faster, seems to work better empirically, and it
48 # matches what was done in Pluribus (Brown and Sandholm. Superhuman AI for
49 # multiplayer poker. Science, 11, 2019).
50 self._average_type = average_type
51
