26// Goofspiel, or the Game of Pure Strategy, is a bidding card game where players
27// are trying to obtain the most points. In, Goofspiel(N,K), each player has bid
28// cards numbered 1..N and a point card deck containing cards numbered 1..N is
29// shuffled and set face-down. There are K turns. Each turn, the top point card
30// is revealed, and players simultaneously play a bid card; the point card is
31// given to the highest bidder or discarded if the bids are equal. For more
32// detail, see: https://en.wikipedia.org/wiki/Goofspiel
33//
34// This implementation of Goofspiel is slightly more general than the standard
35// game. First, more than 2 players can play it. Second, the deck can take on
36// pre-determined orders rather than randomly determined. Third, there is an
37// option to enable the imperfect information variant described in Sec 3.1.4
38// of http://mlanctot.info/files/papers/PhD_Thesis_MarcLanctot.pdf, where only
39// the sequences of wins / losses is revealed (not the players' hands). Fourth,
40// players can play for only K turns (if not specified, K=N by default).
41//
42// The returns_type parameter determines how returns (utilities) are defined:
43// - win_loss distributed 1 point divided by number of winners (i.e. players
44// with highest points), and similarly to -1 among losers
45// - point_difference means each player gets utility as number of points
46// collected minus the average over players.
47// - total_points means each player's return is equal to the number of points
48// they collected.
49//
50// Parameters:
51// "imp_info" bool Enable the imperfect info variant (default: false)
52// "egocentric" bool Enable the egocentric info variant (default: false)
53// "num_cards" int The highest bid card, and point card (default: 13)
54// "num_turns" int The number of turns to play (default: -1, play
55// for the same number of rounds as there are cards)
56// "players" int number of players (default: 2)
57// "points_order" string "random" (default), "descending", or "ascending"
58// "returns_type" string "win_loss" (default), "point_difference", or
59// "total_points".
