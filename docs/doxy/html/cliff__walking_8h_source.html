25// A cliff walking reinforcement learning environment.
26//
27// This is a deterministic environment that can be used to test RL algorithms.
28// Note there are *no illegal moves* in this environment--if the agent is on the
29// gridworld boundary and takes an action which would yield an invalid position,
30// the action is ignored (as if there were walls surrounding the grid world).
31//
32// The player spawns at the bottom left and must reach the goal located on the
33// bottom right. A game is terminal when the player reaches the goal, the
34// maximum episode length has been reached (horizon) or when the player steps
35// off the cliff edge (see figure below). The player receives a reward of -1 for
36// all transitions except when stepping off the cliff, where a reward of -100 is
37// received.
38//
39// Cliff example for height=3 and width=5:
40//
41// | | | | | |
42// | | | | | |
43// | S | x | x | x | G |
44//
45// where `S` is always the starting position, `G` is always the goal and `x`
46// represents the zone of high negative reward to be avoided. For this instance,
47// the optimum policy is depicted as follows:
48//
49// | | | | | |
50// |-->|-->|-->|-->|\|/|
51// |/|\| x | x | x | G |
52//
53// yielding a reward of -6 (minus 1 per time step).
54//
55// See pages 132 of Rich Sutton's book for details:
56// http://www.incompleteideas.net/book/bookdraft2018mar21.pdf
57//
58// Parameters:
59// "height" int rows of the board (default = 4)
60// "width" int columns of the board (default = 8)
61// "horizon" int maximum episode length (default = 100)
