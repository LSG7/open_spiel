23 // An implementation of the 'Coin Game'. Different descriptions of this game
24 // exist with slightly different rules. In particular:
25 // a) "Modeling Others using Oneself in Multi-Agent Reinforcement Learning"
26 // (https://arxiv.org/abs/1802.09640)
27 // b) "Maintaining cooperation in complex social dilemmas using deep
28 // reinforcement learning" (https://arxiv.org/abs/1707.01068)
29 // c) "Learning with Opponent-Learning Awareness"
30 // (https://arxiv.org/abs/1709.04326)
31 // The current implementation follows the description given in a).
32 //
33 // Players live on a a grid, which also contains coins of different colors.
34 // Players can collect coins by moving around and walking into the coin's
35 // square. They can move in all directions or choose not to move at all.
36 // If a player would move outside of the grid or into the square of another
37 // player, they stay where they are.
38 // Each player has a preferred color. They are rewarded for collecting
39 // coins of their own or other players' preference, but punished for collecting
40 // coins that are no one's preference. Players initially only know their own
41 // coin preference. The initial positions of players and coins on the board is
42 // randomized, as well as the players' color preferences.
43 // Players move sequentially, in fixed order, starting with player 0.
