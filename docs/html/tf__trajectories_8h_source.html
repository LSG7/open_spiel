25 // Important note: TF_Trajectories is an unsupported part of OpenSpiel. It has
26 // not tested with CMake and public Tensorflow. We do not anticipate any
27 // problems, but cannot support it officially at this time. We might officially
28 // support it in the future, in which case it would be moved into the core part
29 // of the library.
30 //
31 // This is a class to generate a batch of trajectories entirely in C++ using
32 // Tensorflow policies:
33 // - The graph is created in Python and serialized into a file (using
34 // tf.train.write_graph). See contrib/python/export_graph.py.
35 // - The graph is loaded in C++ and we use the TF C++ API to execute ops.
36 //
37 // This code has been adapted from the Travis Ebesu's blog post:
38 // https://tebesu.github.io/posts/Training-a-TensorFlow-graph-in-C++-API
39
