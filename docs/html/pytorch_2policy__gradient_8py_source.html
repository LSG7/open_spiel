16 r"""Policy Gradient based agents implemented in PyTorch.
17 
18 This class is composed of three policy gradient (PG) algorithms:
19 
20 - Q-based Policy Gradient (QPG): an "all-actions" advantage actor-critic
21 algorithm differing from A2C in that all action values are used to estimate the
22 policy gradient (as opposed to only using the action taken into account):
23 
24  baseline = \sum_a pi_a * Q_a
25  loss = - \sum_a pi_a * (Q_a - baseline)
26 
27 where (Q_a - baseline) is the usual advantage. QPG is also known as Mean
28 Actor-Critic (https://arxiv.org/abs/1709.00503).
29 
30 
31 - Regret policy gradient (RPG): a PG algorithm inspired by counterfactual regret
32 minimization (CFR). Unlike standard actor-critic methods (e.g. A2C), the loss is
33 defined purely in terms of thresholded regrets as follows:
34 
35  baseline = \sum_a pi_a * Q_a
36  loss = regret = \sum_a relu(Q_a - baseline)
37 
38 where gradients only flow through the action value (Q_a) part and are blocked on
39 the baseline part (which is trained separately by usual MSE loss).
40 The lack of negative sign in the front of the loss represents a switch from
41 gradient ascent on the score to descent on the loss.
42 
43 
44 - Regret Matching Policy Gradient (RMPG): inspired by regret-matching, the
45 policy gradient is by weighted by the thresholded regret:
46 
47  baseline = \sum_a pi_a * Q_a
48  loss = - \sum_a pi_a * relu(Q_a - baseline)
49 
50 
51 These algorithms were published in NeurIPS 2018. Paper title: "Actor-Critic
52 Policy Optimization in Partially Observable Multiagent Environment", the paper
53 is available at: https://arxiv.org/abs/1810.09026.
54 
55 - Advantage Actor Critic (A2C): The popular advantage actor critic (A2C)
56 algorithm. The algorithm uses the baseline (Value function) as a control variate
57 to reduce variance of the policy gradient. The loss is only computed for the
58 actions actually taken in the episode as opposed to a loss computed for all
59 actions in the variants above.
60 
61  advantages = returns - baseline
62  loss = -log(pi_a) * advantages
63 
64 The algorithm can be found in the textbook:
65 https://incompleteideas.net/book/RLbook2018.pdf under the chapter on
66 `Policy Gradients`.
67 
68 See open_spiel/python/pytorch/losses/rl_losses_test.py for an example of the
69 loss computation.
70 """

270  # Remove illegal actions, re-normalize probs
271  probs = np.zeros(self._num_actions_num_actions)
272  probs[legal_actions] = policy_probs[0][legal_actions]
273  if sum(probs) != 0:
