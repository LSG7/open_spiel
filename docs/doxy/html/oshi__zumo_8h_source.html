26// Oshi-Zumo is a common benchmark simultaneous move game. Players pay coins
27// each round to bid to move a wrestler, which can move ahead (into opponent's
28// territory) if the bid is won, or back (into player's territory) if the bid
29// is lost. The aim of the original game is to either push the wrestler off the
30// edge of the opponent's side, or end with the wrestler on the opponent's side
31// of the field, resulting in a win. Alesia is a variant that requires the
32// wrestler to be pushed off the side for a win; everything else is a draw.
33//
34// See:
35// - M. Buro 2003, "Solving the Oshi-Zumo game".
36// - Bosansky et al 2016, "Algorithms for Computing Strategies in Two-Player
37// Simultaneous Move Games".
38// - Also called Alesia (slight variant) in Perolat et al. 2016,
39// "Softened Approximate Policy Iteration for Markov Games".
40//
41// Parameters:
42// "alesia" bool draw if wrestler is not pushed off (default: false)
43// "coins" int number of coins each player starts with (default: 50)
44// "size" int size of the field (= 2*size + 1) (default: 3)
45// "horizon" int max number of moves before draw (default: 1000)
46// "min_bid" int minimum bid at each turn (default: 0)
