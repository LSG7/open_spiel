15 // Mean Field Crowd Modelling Game.
16 //
17 // This game corresponds to the "Beach Bar Process" defined in section 4.2 of
18 // "Fictitious play for mean field games: Continuous time analysis and
19 // applications", Perrin & al. 2019 (https://arxiv.org/abs/2007.03458).
20 //
21 // In a nutshell, each representative agent evolves on a circle, with {left,
22 // neutral, right} actions. The reward includes the proximity to an imagined bar
23 // placed at a fixed location in the circle, and penalties for moving and for
24 // being in a crowded place.

49 // Game state.
50 // The high-level state transitions are as follows:
51 // - First game state is a chance node where the initial position on the
52 // circle is selected.
53 // Then we cycle over:
54 // 1. Decision node with actions {left, neutral, right}, represented by integers
55 // 0, 1, 2. This moves the position on the circle.
56 // 2. Mean field node, where we expect that external logic will call
57 // DistributionSupport() and UpdateDistribution().
58 // 3. Chance node, where one of {left, neutral, right} actions is externally
59 // selected.
60 // The game stops after a non-initial chance node when the horizon is reached.
