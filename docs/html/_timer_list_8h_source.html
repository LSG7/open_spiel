10 /*
11  TimerList consists of a number of groups, one for each piece
12  of the code being timed (ABsearch etc).
13 
14  Each group corresponds to something that should be timed at
15  multiple AB depths, i.e. cards played. The first card of a
16  new game is number 48, and the last card is number 0.
17 
18  The AB timer is special, as the AB functions are recursive
19  and so their timing includes not only the other functions they
20  contain, but also their own recursive calls at lower depths.
21  The AB timer group must be the first one.
22 
23  The object calculates an approximation to exclusive function
24  times, so it is a "poor man's profiler".
25 
26  For AB, first the times at depth-1 are subtracted out, and then
27  the times for all calls at the same depth are subtracted out.
28  This still leaves the overhead of the timing itself. As an
29  approximation, there is one timing overhead left for each
30  function, and it is on the order of the execution time of
31  Evaluate(), which is a very fast function.
32 
33  TIMER_START and TIMER_END are macros for bracketing code
34  to be timed, so
35 
36  TIMER_START(TIMER_NO_AB, depth);
37  ABsearch(...);
38  TIMER_END(TIMER_NO_AB, depth);
39 
40  This avoids the tedious #ifdef's at every place of a timer.
41  */
