29 // A vanilla Monte Carlo Tree Search algorithm.
30 //
31 // This algorithm searches the game tree from the given state.
32 // At the leaf, the evaluator is called if the game state is not terminal.
33 // A total of max_simulations states are explored.
34 //
35 // At every node, the algorithm chooses the action with the highest PUCT value
36 // defined as: `Q/N + c * prior * sqrt(parent_N) / N`, where Q is the total
37 // reward after the action, and N is the number of times the action was
38 // explored in this position. The input parameter c controls the balance
39 // between exploration and exploitation; higher values of c encourage
40 // exploration of under-explored nodes. Unseen actions are always explored
41 // first.
42 //
43 // At the end of the search, the chosen action is the action that has been
44 // explored most often. This is the action that is returned.
45 //
46 // This implementation supports sequential n-player games, with or without
47 // chance nodes. All players maximize their own reward and ignore the other
48 // players' rewards. This corresponds to max^n for n-player games. It is the
49 // norm for zero-sum games, but doesn't have any special handling for
50 // non-zero-sum games. It doesn't have any special handling for imperfect
51 // information games.
52 //
53 // The implementation also supports backing up solved states, i.e. MCTS-Solver.
54 // The implementation is general in that it is based on a max^n backup (each
55 // player greedily chooses their maximum among proven children values, or there
56 // exists one child whose proven value is Game::MaxUtility()), so it will work
57 // for multiplayer, general-sum, and arbitrary payoff games (not just win/loss/
58 // draw games). Also chance nodes are considered proven only if all children
59 // have the same value.
60 //
61 // Some references:
62 // - Sturtevant, An Analysis of UCT in Multi-Player Games, 2008,
63 // https://web.cs.du.edu/~sturtevant/papers/multi-player_UCT.pdf
64 // - Nijssen, Monte-Carlo Tree Search for Multi-Player Games, 2013,
65 // https://project.dke.maastrichtuniversity.nl/games/files/phd/Nijssen_thesis.pdf
66 // - Silver, AlphaGo Zero: Starting from scratch, 2017
67 // https://deepmind.com/blog/article/alphago-zero-starting-scratch
68 // - Winands, Bjornsson, and Saito, Monte-Carlo Tree Search Solver, 2008.
69 // https://dke.maastrichtuniversity.nl/m.winands/documents/uctloa.pdf

79 // Abstract class representing an evaluation function for a game.
80 // The evaluation function takes in an intermediate state in the game and
81 // returns an evaluation of that state, which should correlate with chances of
82 // winning the game for player 0.
83 class Evaluator {
84  public:
85  virtual ~Evaluator() = default;
86 
87  // Return a value of this state for each player.
88  virtual std::vector<double> Evaluate(const State& state) = 0;
89 
90  // Return a policy: the probability of the current player playing each action.
91  virtual ActionsAndProbs Prior(const State& state) = 0;
92 };

119  double total_reward = 0; // Total reward passing through this node.
