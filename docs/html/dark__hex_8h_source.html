28 // Dark Hex (Some versions also called Phantom Hex or Kriegspiel Hex) is an
29 // imperfect information version of the classic game of Hex. Players are not
30 // exposed to oppsite sides piece information. Only a refree has the full
31 // information of the board. When a move fails due to collision/rejection the
32 // player gets some information of the cell (i.e. stone exists), and is allowed
33 // to make another move until success.
34 //
35 // There are slightly different versions of the game exists depending on the
36 // level of information being exposed to the opponent and what happens in the
37 // event of an attempt to move to an occupied cell. We have two different
38 // versions of Dark Hex (Phantom Hex) implemented:
39 // - Classical Dark Hex (cdh)
40 // Player:
41 // -> Replays after the attempt to move to an occupied cell
42 // (Rejection)
43 // - Abrupt Dark Hex (adh)
44 // Player:
45 // -> No replay after the attempt to move to an occupied cell
46 // (Collision)
47 //
48 // For classical dark hex we do allow specifying 'obstype'. It specifies if the
49 // player is exposed to number of turns that has passed or not.
50 //
51 // Common phantom games include Kriegspiel (Phantom chess), e.g. see
52 // https://en.wikipedia.org/wiki/Kriegspiel_(chess), and Phantom Go.
53 // See also http://mlanctot.info/files/papers/PhD_Thesis_MarcLanctot.pdf, Ch 3.
54 //
55 // Parameters:
56 // "gameversion" string Which version of the game to activate
57 // (default "cdh") ['cdh', 'adh']
58 // "obstype" string If the player is informed of the
59 // number of moves attempted
60 // (default "reveal-nothing") ['reveal-nothing', 'reveal-numturns']
61 //
62 // "num_cols" int Number of columns on the hex board
63 // (default 3)
64 // "num_rows" int Number of the rows on the hex board
65 // (default 3)
