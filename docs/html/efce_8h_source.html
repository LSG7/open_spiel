26 // The implementations of the metrics assemble the extended game described in
27 // von Stengel and Forges 2008, Definition 2.2. Then, the incentive to deviate
28 // to a best response is computed by running NashConv on this auxiliary game.
29 //
30 // The EFCE extended game modifies the original game in the following ways:
31 // - An internal variable is kept to determine if the player has deviated.
32 // If so, recommendations are no longer given.
33 // - If config.deterministic and config.convert are both false, the game
34 // starts with a "Monte Carlo" chance node that corresponds to different
35 // samplings of deterministic joint policies. The number of samples (and
36 // corresponding accuracy) is determined by config.num_samples. If either
37 // config.deterministic is true or config.convert is true, this chance node
38 // will not exist.
39 // - A "joint policy" chance node that corresponds to choosing a joint policy
40 // from the correlation device.
41 // - Information state keys are modified to include the recommendations
42 // received at the current information state and ones received up to this
43 // information state, i.e. the sequence of recommendations. New
44 // recommendations stop getting appended once the player chooses an action
45 // that does not match the recommendation.
46 //
47 // In addition, a specific tabular policy is made so as to map the policies
48 // in this new game back to the original game.
