15// Mean Field Crowd Modelling Game in 2d.
16//
17// This game corresponds to a 2D "Beach Bar Process" defined in section 4.2 of
18// "Fictitious play for mean field games: Continuous time analysis and
19// applications", Perrin & al. 2019 (https://arxiv.org/abs/2007.03458).
20//
21// In a nutshell, each representative agent evolves on a 2d torus, with {down,
22// left, neutral, right, up} actions. The reward includes the proximity to an
23// imagined bar placed at a fixed location in the torus, and penalties for
24// moving and for being in a crowded place.

71// Game state.
72// The high-level state transitions are as follows:
73// - First game state is a chance node where the initial position on the
74// torus is selected.
75// Then we cycle over:
76// 1. Decision node with actions {down, left, neutral, right, up}, represented
77// by integers 0, 1, 2, 3, 4. This moves the position on the torus.
78// 2. Mean field node, where we expect that external logic will call
79// DistributionSupport() and UpdateDistribution().
80// 3. Chance node, where one of {down, left, neutral, right, up} actions is
81// externally selected.
82// The game stops after a non-initial chance node when the horizon is reached.

85 // forbidden_states, initial_distribution and positional_reward are formated
86 // like '[int|int;...;int|int]'. Example : "[]" or "[0|0;0|1]".

88 // '[float;...;float]'. Example : "[]" or "[0.5;0.5]"
