26 // Returns a distribution over states at the information state containing the
27 // specified state given the opponents' policies. That is, it returns
28 // Pr(h | s, \pi_{-i}) by normalizing the opponents' reach probabilities over
29 // all h \in s, as described in Section 3.2 of Srinivasan et al. 2018
30 // https://arxiv.org/abs/1810.09026. Computing this distribution relies strongly
31 // on the fact that InformationStateString must abide by perfect recall.
32 //
33 // This is a game-independent implementation that does a breadth-first search
34 // from the start of the game to enumerate all possible histories consistent
35 // with the information state, trimming out histories as they become invalid.
36 // As such, it may not be very fast.
37 //
38 // The returned vectors have an arbitrary ordering, and will include
39 // zero-probability histories if there are any. If the probability of reaching
40 // the information state under the given policy is zero (e.g. the Bayes
41 // normalization term is zero) then a uniform random distribution is returned
42 // instead.
43 //
44 // Note: currently only works for turn-based games of imperfect information,
45 // and does not work with kSampledStochastic chance modes.
46 HistoryDistribution GetStateDistribution(const State& state,
47  const Policy& opponent_policy);

53 // Incrementally builds the state distribution vectors. Must be called at each
54 // state in a trajectory. All of the states should correspond to the same
55 // information state (i.e. all states should have identical
56 // InformationStateString values, although this is not doublechecked). If
57 // previous is empty, calls the non-incremental version. This must be called for
58 // each state in order, starting from the first non-chance node, or it will be
59 // wrong.
60 // Takes ownership of previous.
