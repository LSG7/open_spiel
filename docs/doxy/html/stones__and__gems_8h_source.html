25// A simplified version of a mixture of various common stone and gem games.
26// For details of the original games, see “Boulder Dash” (First Star Software,
27// 1984) and Emerald Mines (Kingsoft, 1987)
28// Brief:
29// - The agent's goal is to go through the exit door.
30// - The agent can move up, down, left, right, or stand still.
31// - In order to open the exit, a minimum number of gems need to be
32// collected.
33// - Objects are suspended by dirt, or otherwise fall. The agent can move in
34// all directions,
35// and can remove dirt by walking over it.
36// - The agent can push stones horizontally if there is room to do so.
37// - The agent can die if objects fall on top of him, or if he collides with
38// enemies.
39// - Fireflies try to move clockwise.
40// - Butterflies try to move counter-clockwise.
41// - Both fireflies and butterflies explode if stones fall on them.
42// - Butterflies can drop diamonds upon being killed.
43// - Oranges move either up, left, down, or right. If they hit another
44// object, they randomly start to move in another direction.
45// The agent will die if he runs into an Orange.
46// - Magic walls convert diamonds to stones, and stones to diamonds, but
47// only when activated. Magic walls can be activated by dropping a stone
48// through it, and will stop being active after a set amount of time.
49// - Blobs grow randomly. If trapped, they become diamonds. If they grow too
50// large, they turn into stones.
51// - Keys and gates come in 4 colours: red, blue, green, and yellow. The
52// gates remain closed until the agent collects the corresponding key.
53// Once open, the agent can pass through.
54// - Nuts can fall down like diamonds and stones. If a stones falls on it,
55// it will open to reveal a diamond.
56// - Bombs will explode if they fall onto an object, or a stone falls on top
57// of it. The agent can push bombs.
58//
59// NOTE: Formatted levels from various popular stone and gem games such as
60// Boulder Dash, as well as various
61// simple RL levels can be found here
62// https://github.com/tuero/stone_gems_levels
63//
64// Parameters:
65// "magic_wall_steps" int steps magic walls remain active once
66// turned on (default = 140)
67// "blob_chance" int chance out of 256 each blob will
68// spawn another (default = 20)
69// "blob_max_percentage" double maximum blob growth size, as
70// percentage of map (default = 0.16)
71// "rng_seed" int seed for internal rng (default = 0)
72// "grid" std::string string representing map (see
73// kDefaultGrid below)
74//
75// Grid parameter specification
76// - The grid string parameter is a pipe-separated (|) string representing
77// the map
78// - The first line should contain the # of cols, # of rows, max_steps, and
79// gems required
80// - The following lines represent the rows, with elements column separated
81// - Item values are the cell type ints given by HiddenCellType (see below)
82

194// Element entities, along with properties
195struct Element {
196 HiddenCellType cell_type;
197 VisibleCellType visible_type;
198 int properties;
199 char id;
200 bool has_updated;
201
202 Element()
203 : cell_type(HiddenCellType::kNull),
204 visible_type(VisibleCellType::kNull),
205 properties(0),
206 id(0),
207 has_updated(false) {}
208
209 Element(HiddenCellType cell_type, VisibleCellType visible_type,
210 int properties, char id)
211 : cell_type(cell_type),
212 visible_type(visible_type),
213 properties(properties),
214 id(id),
215 has_updated(false) {}
216
217 bool operator==(const Element& rhs) const {
218 return this->cell_type == rhs.cell_type;
219 }
220
221 bool operator!=(const Element& rhs) const {
222 return this->cell_type != rhs.cell_type;
223 }
224};
