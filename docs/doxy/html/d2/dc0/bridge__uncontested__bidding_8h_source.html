20// Uncontested bridge bidding. A two-player purely cooperative game.
21//
22// The full game of contract bridge is played by four players in two
23// partnerships; it consists of a bidding phase followed by a play phase. The
24// bidding phase determines the contract for the play phase. The contract has
25// four components:
26// - Which of the four players is the 'declarer'. (The first play is made by
27// the player on declarer's left. Declarer's partner (the 'dummy') then
28// places their cards face-up for everyone to see; their plays are chosen
29// by declarer.)
30// - The trump suit (or no-trumps).
31// - The level, i.e. the trick target for the declaring partnership.
32// - Whether the contract is doubled or redoubled (increasing the stakes).
33//
34// In 'uncontested bidding', we simplify the game in two ways:
35// 1. Only one partnership may bid during the auction phase (hence
36// 'uncontested').
37// 2. Rather than play out the play phase, we generate several (e.g. 10)
38// layouts of the opponents' cards, solve for the number of tricks that
39// would be taken with perfect perfect-information play by both sides
40// on each deal, and use the average score over these deals. (This
41// perfect information solution is called 'double dummy', because it is
42// equivalent to one player of each partnerships being 'dummy' in the sense
43// described above).
44//
45// Since the other partnership has no actions available, this is
46// a two-player cooperative game. It is widely used by partnerships
47// to practice their bidding. See for example this on-line tool:
48// http://www.bridgebase.com/help/v2help/partnership_bidding.html
49// Or these pre-constructed hands:
50// http://rpbridge.net/rpbp.htm (here the scores are generated using human
51// judgement rather than the automated procedure given above).
52//
53// We support two variations:
54// 1. Any deal permitted, auction starts normally.
55// In this variant, WBridge5 scores +95.1 absolute, std err 2.2
56// Its relative score (compared to the best-possible score on each deal)
57// is -68.8, std err 1.3 (both results from n=8750 deals).
58// 2. First player is dealt a hand suitable for a 2NT opening (i.e. a bid
59// of 8 tricks with no trumps), and is forced to bid 2NT.
60// A 2NT opening is typically played as showing a very strong balanced
61// hand. 'Balanced' means that the distribution of cards between the
62// suits is 4-3-3-3, 4-4-3-2, or 5-3-3-2 (some might also incude some
63// 6-3-2-2 or 5-4-2-2 hands, but we do not).
64// Strength is typically measured using 'high card points' (A=4, K=3, Q=2,
65// J=1). A 2NT opening on this scale might be 20-22, 20-21, 21-22, or
66// similar depending on agreement. We use 20-21, in line with the
67// 'Standard American Yellow Card' system:
68// http://web2.acbl.org/documentlibrary/play/SP3%20(bk)%20single%20pages.pdf
69// Expert players may adjust hand valuation up or down slightly depending
70// on the location of their high cards; we do not attempt to replicate
71// this.
72//
73// The 2NT variant is both a smaller game, and also a fairer comparison
74// with existing bots, since in practice auctions which start with 2NT are
75// almost always uncontested. This means that bidding is generally conducted
76// without worrying that the opponents might bid. This is in contrast to the
77// full game where many bids are designed in anticipation of the possibility
78// that the opponents may bid - a constraint that we do not have in this game.
79//
80// We treat the initial deal as a single sampled stochastic chance event; that
81// is, the game tree has a single chance event with a single possible outcome,
82// but when applying this outcome, the game state evolves stochastically,
83// reflecting the full deal that has taken place.
84//
85// We could have explicit chance in case (1), e.g. with one chnce node for each
86// card being dealt, but this would be hard in case (2), and we choose to
87// treat both consistently.
88//
89// The score for player 0 will always be the raw point score for the contract
90// reached. If the parameter `relative_scoring` is set to true, then the score
91// for player 1 will be the score relative to the best-scoring of the possible
92// contracts (so 0 if the contract reached is the best-scoring contract,
93// otherwise negative).

119class Deal {
120 public:
121 Deal() { std::iota(std::begin(cards_), std::end(cards_), 0); }
122 void Shuffle(std::mt19937* rng, int begin = 0, int end = kNumCards) {
123 for (int i = begin; i < end - 1; ++i) {
124 // We don't use absl::uniform_int_distribution because it behaves
125 // differently in different versions of C++, and we want reproducible
126 // tests.
127 int j = i + (*rng)() % (end - i);
128 std::swap(cards_[i], cards_[j]);
129 }
130 }
131 Deal(const std::array<int, kNumCards>& cards) : cards_(cards) {}
132 int Card(int i) const { return cards_[i]; }
133 int Suit(int i) const { return cards_[i] % kNumSuits; }
134 int Rank(int i) const { return cards_[i] / kNumSuits; }
135 std::string HandString(int begin, int end) const;
136
137 private:
138 std::array<int, kNumCards> cards_; // 0..12 are West's, then E, N, S
139};
