35 // How to average the strategy. The 'simple' type does the averaging for
36 // player i + 1 mod num_players on player i's regret update pass; in two players
37 // this corresponds to the standard implementation (updating the average
38 // policy at opponent nodes). In n>2 players, this can be a problem for several
39 // reasons: first, it does not compute the estimate as described by the
40 // (unbiased) stochastically-weighted averaging in chapter 4 of Lanctot 2013
41 // commonly used in MCCFR because the denominator (important sampling
42 // correction) should include all the other sampled players as well so the
43 // sample reach no longer cancels with reach of the player updating their
44 // average policy. Second, if one player assigns zero probability to an action
45 // (leading to a subtree), the average policy of a different player in that
46 // subtree is no longer updated. Hence, the full averaging does not update the
47 // average policy in the regret passes but does a separate pass to update the
48 // average policy. Nevertheless, we set the simple type as the default because
49 // it is faster, seems to work better empirically, and it matches what was done
50 // in Pluribus (Brown and Sandholm. Superhuman AI for multiplayer poker.
51 // Science, 11, 2019).
52 enum class AverageType {
53  kSimple,
54  kFull,
55 };
