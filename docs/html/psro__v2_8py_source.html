15 """Modular implementations of the PSRO meta algorithm.
16 
17 Allows the use of Restricted Nash Response, Nash Response, Uniform Response,
18 and other modular matchmaking selection components users can add.
19 
20 This version works for N player, general sum games.
21 
22 One iteration of the algorithm consists of:
23 
24 1) Computing the selection probability vector (or meta-strategy) for current
25 strategies of each player, given their payoff.
26 2) [optional] Generating a mask over joint policies that restricts which policy
27 to train against, ie. rectify the set of policies trained against. (This
28 operation is designated by "rectify" in the code)
29 3) From every strategy used, generating a new best response strategy against the
30 meta-strategy-weighted, potentially rectified, mixture of strategies using an
31 oracle.
32 4) Updating meta game matrix with new game results.
33 
34 """

53  """A general implementation PSRO.
54 
55  PSRO is the algorithm described in (Lanctot et Al., 2017,
56  https://arxiv.org/pdf/1711.00832.pdf ).
57 
58  Subsequent work regarding PSRO's matchmaking and training has been performed
59  by David Balduzzi, who introduced Restricted Nash Response (RNR), Nash
60  Response (NR) and Uniform Response (UR).
61  RNR is Algorithm 4 in (Balduzzi, 2019, "Open-ended Learning in Symmetric
62  Zero-sum Games"). NR, Nash response, is algorithm 3.
63  Balduzzi et Al., 2019, https://arxiv.org/pdf/1901.08106.pdf

265  def _restrict_target_training(self,
266  current_player,
267  ind,
268  total_policies,
269  probabilities_of_playing_policies,
270  restrict_target_training_bool,
271  epsilon=1e-12):
272  """Rectifies training.
273 
274  Args:
275  current_player: the current player.
276  ind: Current strategy index of the player.
277  total_policies: all policies available to all players.
278  probabilities_of_playing_policies: meta strategies.
279  restrict_target_training_bool: Boolean specifying whether to restrict
280  training. If False, standard meta strategies are returned. Otherwise,
281  restricted joint strategies are returned.
282  epsilon: threshold below which we consider 0 sum of probabilities.
283 
284  Returns:
285  Probabilities of playing each joint strategy (If rectifying) / probability
286  of each player playing each strategy (Otherwise - marginal probabilities)
287  """
288  true_shape = tuple([len(a) for a in total_policies])
289  if not restrict_target_training_bool:
290  return probabilities_of_playing_policies
291  else:
292  kept_probas = self._rectifier_rectifier(
293  self, current_player, ind)
294  # Ensure probabilities_of_playing_policies has same shape as kept_probas.
295  probability = probabilities_of_playing_policies.reshape(true_shape)
296  probability = probability * kept_probas
297  prob_sum = np.sum(probability)
298 
299  # If the rectified probabilities are too low / 0, we play against the
300  # non-rectified probabilities.
301  if prob_sum <= epsilon:
302  probability = probabilities_of_playing_policies
303  else:
304  probability /= prob_sum
305 
306  return probability
